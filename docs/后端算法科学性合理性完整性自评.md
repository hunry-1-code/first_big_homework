# 后端算法科学性、合理性与完整性自评

> 评估日期：2026-07-13  
> 评估对象：`D:\newbigwork\v2\first_big_homework` 当前 `branchhu` 分支  
> 评估范围：后端数据采集、处理、分析、事件生成、风险与传播分析、LLM 能力及工程保障  
> 评估原则：工程测试通过不等于真实世界准确率已经得到证明

## 一、总体结论

当前后端已经不再是只展示流程的雏形。数据模型、任务状态、失败降级、算法版本、证据字段、持久化快照和 API 契约已形成较完整闭环；内容分析、事件聚合、热点、情感、生命周期、风险、传播、Top10 和 LLM 等模块均有可运行实现，而不是前端 mock。

但从科学验证角度，系统仍属于“工程可运行、方法基本合理、经验参数较多、真实标注评估不足”的阶段。现在能够证明的是：

- 主要算法路径有确定输入输出并可重复运行；
- 已知回归问题有自动化测试保护；
- 三个热榜来源和一次最小 LLM 调用在本次验证时可用；
- 失败可以降级、分类和持久化；
- 前端需要的主要 API 数据结构可以稳定返回。

目前不能证明的是：

- 事件聚合在不同领域上的准确率、召回率达到某个数值；
- 情感分类、生命周期、风险或传播边具有经过代表性标注集验证的可靠精度；
- 热度分数等同于客观社会影响力；
- 可疑信息风险分数可以裁定新闻真假；
- LLM 生成的地点、人物、起因、摘要和问答答案必然正确；
- 一次真实接口成功代表第三方平台长期稳定。

按本文 1～5 分标准，13 个模块的平均自评分为：

- 科学性：约 **3.3/5**
- 工程合理性：约 **3.5/5**
- 完整性：约 **3.9/5**

这意味着系统已经具备课程项目或原型产品向成熟后端演进的基础，但若要对外宣称“智能研判准确”“事实核查可靠”或“传播路径真实还原”，仍需标注数据、人工评估和线上监控证据。

## 四、多平台采集与热榜适配

### 4.1 当前实现

采集层通过统一 `CrawlRequest` 和 `RawDocument` 契约连接百度搜索/热榜、知乎搜索/热榜、微博搜索/热搜、B站、小红书、抖音、RSS 和本地 sample。HTTP 请求包含超时、响应大小限制、平台级配置和错误类型。第三方失败被分类，不将完整远端响应直接传播到业务接口。

今日榜单直接使用 `weibo_hot`、`baidu_hot` 和 `zhihu_hot`。2026-07-13 的低频验证中，三者各返回 1 条有效 RawDocument，预处理状态均为 success。

### 4.2 科学性

采集本身不是推断算法，科学性主要体现在抽样边界和数据可追溯性。统一契约、保留原始字段、记录平台和时间，是后续分析可复现的必要条件。当前实现满足基本要求。

不足在于多平台数据不是随机样本。热榜是平台算法筛选结果，搜索结果也受到排名、个性化、接口可见范围和平台审核影响。因此系统看到的是“可访问的平台样本”，不能直接代表全体公众意见。

### 4.3 合理性与完整性

以直接平台热榜作为首页 Top10 来源，比从所有爬取文章中“大水漫灌”更符合用户对“今日热点”的预期。单源失败隔离和缓存陈旧标识也符合生产使用。

当前缺少长期可用性统计、适配器结构变化报警、来源采样偏差说明和平台速率预算。一次成功不能证明持续成功。

### 4.4 下一步实验

连续 30 天按固定时间低频采集，记录每个适配器的成功率、空响应率、P50/P95 延迟、结构错误率、榜单条目重复率和来源覆盖变化。

## 五、正文提取、清洗、质量与去重

### 5.1 当前实现

预处理采用分阶段管线：

1. 字段标准化和发布时间解析；
2. 正文提取及 fallback；
3. 噪声清理；
4. 文本质量、广告和置信度评分；
5. URL、标题、内容哈希、SimHash/Jaccard 和可选 BGE 去重；
6. jieba 分词与失败降级；
7. 保存阶段状态、版本、警告和 NLP 权重。

### 5.2 科学性

“确定性规则先处理、语义模型处理边界样本”的分层方式是合理的。URL/哈希适合精确重复，Jaccard/SimHash 适合近重复，BGE 可补充语义相似，这些方法解决的问题不同，不宜互相替代。

质量分数本质是工程加权，不是经过监督学习得到的真实性或新闻质量概率。将它作为 NLP 权重和垃圾降权是合理的，将它解释为内容可信度则不科学。当前代码已把文本质量和来源可信度分离，这是正确边界。

### 5.3 完整性

阶段结果包含 success/degraded/failed、版本和警告；正文提取失败不会伪造正常正文；BGE 不可用时保留 pending/降级证据。整体完整性较高。

仍需验证不同页面模板、短视频文本、论坛回答、表格新闻、广告软文和机器转载的提取质量。

### 5.4 下一步实验

建立至少 500 篇多平台正文标注集，评价正文字符级 precision/recall、空正文率、广告误判率、重复对 precision/recall，并按平台分别报告。

## 六、分词、TF-IDF、关键词、实体与 BGE 表示

### 6.1 当前实现

内容表示包含：

- jieba 分词和通用 fallback；
- 标题/正文加权 TF-IDF；
- 实体、主题和查询词来源；
- BGE/Sentence Transformer 可选向量；
- LLM 小批量多维关键词；
- LLM 失败后逐篇回退 TF-IDF/实体/主题；
- 查询词以 `source=query` 降权保留；
- 关键词持久化为 JSON-safe 字典；
- 关键词附带 score、sentiment 和 entity type。

### 6.2 科学性

TF-IDF 对局部语料中的区分词有效，BGE 对语义相近但词面不同的文本更有优势。二者与实体、时间组合，比依赖单一余弦相似度更稳健。负余弦不再被线性映射为正相似，也是正确修正。

LLM 关键词可以改善可读性和实体类型，但其分数并非统计意义上的概率。LLM 与 TF-IDF 混用时必须保留来源和降级信息，当前实现基本做到。

### 6.3 合理性与完整性

小批量 LLM、逐篇回退和 BGE 开关兼顾成本与可用性。查询词保留但降权，也比直接删除或置顶更合理。

主要不足是 jieba 专业词典、别名归一、多语言、英文缩写、电影/体育/金融实体词典尚未系统评估；BGE 模型也未在本项目事件对数据上校准。

### 6.4 下一步实验

建立文章—关键词人工标注集，评价 Precision@5、Recall@10、nDCG 和实体类型准确率；建立同事件/异事件文本对，比较 TF-IDF、BGE 和混合表示的 ROC-AUC、PR-AUC。

## 七、LDA 主题发现与主题命名

### 7.1 当前实现

热点发现使用 TF-IDF/计数表示和 LDA，配置主题数上限、迭代次数、随机种子、主题词数量、主题多样性、小主题比例和低置信阈值。小语料、单文档、空词表都有明确降级。主题名称可由规则或 LLM 生成，并保存命名方法与置信度。

### 7.2 科学性

LDA 是经典概率主题模型，适合中等规模、词袋假设近似成立的语料，具有较强可解释性。但网络热点文本往往短、实体密集、主题快速变化，词袋假设和固定主题数可能不够适配。小语料中 LDA 主题容易受高频词和随机波动影响。

当前系统没有把 LDA 主题直接当作正式事件，而是后续经过事件聚合和发布，这降低了主题漂移造成的业务风险。

### 7.3 完整性

配置、随机种子、警告、主题词、文章归属和运行记录均被保存，工程完整性较高。缺少的是主题一致性指标、主题稳定性比较和人工可解释性评价。

### 7.4 下一步实验

对不同事件领域计算 c_v/NPMI coherence、主题多样性和不同随机种子的稳定性；由人工评审主题纯度、命名准确度和“一个主题对应多个事件”的比例。

## 八、事件聚合、合并候选与相似事件

### 8.1 当前实现

事件聚合组合 BGE、TF-IDF、实体和时间证据，默认权重为 0.55/0.20/0.15/0.10。创建阈值 0.40、加入阈值 0.55，中间区间标记 ambiguous。第一轮后会重新处理模糊样本，并以 move margin 约束二次重分配；实体硬冲突可阻止错误移动。时间相似采用半衰期式衰减。

正式发布具有幂等、事件匹配、合并候选、相似事件、后处理和持久化快照。

### 8.2 科学性

多视角相似度和可用特征权重重归一化是合理的。相比“只按标题余弦”或“单遍按输入顺序聚类”，二次分配和移动裕量能减少顺序依赖。时间半衰期比硬截止更符合事件延续。

权重和阈值仍是经验参数。没有标注事件簇时，不能证明 0.55/0.40 是最优值。不同领域的实体密度和事件时长差异较大，共用参数可能导致体育赛事、电影宣发和长期政策议题表现不同。

### 8.3 完整性

运行、文章分配、簇、候选、版本、告警、正式发布和后处理形成闭环，完整性较高。真实评价仍停留在小规模案例和合成测试。

### 8.4 下一步实验

标注至少 2,000 对同事件/异事件文章和 200 个真实事件簇，报告 pairwise precision/recall/F1、B-cubed、ARI、簇纯度、过度合并率和过度拆分率，并按自然灾害、娱乐、经济、政策等领域切分。

## 九、今日 Top10 标题归一化与 RRF

### 9.1 当前实现

今日 Top10 使用独立 Run/Item 表，不直接污染正式 Event。标题经过 NFKC、话题符号、外层括号、重复空白和热度尾缀处理。相同规范键的多来源名次通过 Reciprocal Rank Fusion：

```text
score(item) = Σ 1 / (k + rank_source)
```

同一来源只贡献最佳名次，默认 `k=60`。结果支持 TTL、失败来源诊断、管理员刷新、租约任务和逐条事件补全。

### 9.2 科学性

RRF 的优势是只依赖名次，不要求微博、百度和知乎的原始热度数值可比。对异构排名融合，这是比直接相加热度值更科学的选择。

其局限是来源等权和标题等价判断。标题规范化能合并显然相同的表达，但对同义改写、简称和同名不同事件仍可能错合并或漏合并。当前实现没有用真实人工排序验证 `k=60` 或来源等权。

### 9.3 合理性与完整性

首页先快速展示热榜快照，再异步补齐正式事件，是用户体验和计算成本之间的合理分层。`/api/hotspots/today` 与正式 `/api/hotspots` 语义独立，避免把原始热词冒充分析后的事件。

三源真实低频验证已成功，但仍需长期来源稳定性和最终榜单相关性评估。

### 9.4 下一步实验

连续采集 30～90 天，由人工对每日候选做相关性和重复判断，比较 RRF、Borda、最佳名次和来源加权方案的 nDCG@10、重复率、来源覆盖率和榜单稳定性。

## 十、情感分析与事件级聚合

### 10.1 当前实现

文章情感主路径使用 OpenAI-compatible LLM 返回正面/中立/负面、分数、置信度和理由；缓存命中优先。模型不可用时降级 SnowNLP，并记录 `SNOWNLP_FALLBACK` 与 `DOMAIN_MISMATCH_FALLBACK`，限制置信度。事件级聚合考虑文章质量、重复/垃圾降权和传播热度，传播加权上限为 1.5。摘要从同一持久化快照确定性生成。

### 10.2 科学性

LLM 在中文语境、实体和上下文理解上通常优于通用 SnowNLP，但没有项目标注集时仍不能证明准确率。SnowNLP 的训练领域与网络舆情不完全一致，将其限制为低置信降级是合理的。

事件情感按质量和代表性加权，比简单文章计数更科学；限制极端互动文章权重，也能防止单条爆款控制整体结论。但加权上限 1.5 仍是经验选择。

### 10.3 完整性

文章结果、事件快照、平台/日期聚合、缓存、模型名、算法版本、理由和警告较完整。缺少讽刺、反问、转述、混合情绪和不同领域的系统评价。

### 10.4 下一步实验

建立不少于 3,000 篇多领域中文舆情标注集，双人标注并计算一致性；报告 macro-F1、各类 precision/recall、Brier score 或 ECE，并专门评价讽刺、引用和短文本。

## 十一、热度计算

### 11.1 当前实现

热度组合独立报道量、24/48 小时增长、平台覆盖、`log1p` 互动强度和时间半衰期；重复、垃圾和质量权重参与计算。核心热度与传播热度按配置合成，缺失传播数据时记录 warning。快照保存原始统计、分项得分、权重、时间置信度、公式版本和计算时间。

### 11.2 科学性

使用 `log1p` 抑制极端互动量、用指数半衰期表达新鲜度、区分报道数量与平台覆盖，都是合理的稳健处理。

但热度是人为定义的排序函数，不是自然界存在的真实量。当前权重没有通过点击、搜索趋势、人工热度排名或后续传播量进行学习或校准。因此热度分数适合系统内排序，不宜解释为“社会影响力 83.2 分”具有绝对尺度。

### 11.3 完整性

正式热点和搜索发布事件都创建统一快照，解决了历史上只有 Event 数字、没有计算证据的问题。时间缺失、传播缺失和事件离榜也有状态。

### 11.4 下一步实验

选择人工每日热点排序、搜索指数、平台阅读/互动或后续 24 小时报道增长作为目标，评价 Spearman/Kendall 相关、nDCG@10，并用时间切分调参，避免未来信息泄漏。

## 十二、生命周期阶段判断

### 12.1 当前实现

生命周期采用潜伏期、成长期、高潮期、消退期四阶段。使用时间点数量、相对峰值、近期斜率、增长/下降持续性和数据覆盖度；数据不足返回 `data_insufficient` 和低置信度。阶段转移具有单调约束和证据字段，结果在事件更新或热度快照更新时持久化，详情查询只读。

### 12.2 科学性

相对峰值和斜率比固定“报道数大于某值就是高潮期”更适合不同规模事件。状态机和滞回可抑制日间噪声造成的阶段跳动。

但它仍是规则分类器，不是真正预测未来的模型。“生命周期预测”这一名称容易高估能力；更准确的表述是“基于当前序列的阶段估计”。再爆发、多峰事件、长尾政策和电影宣发可能违反单调四阶段假设。

### 12.3 完整性

阶段、状态、置信度、证据、更新时间和 API 枚举已经闭环。缺少真实事件日序列标注和多峰周期建模。

### 12.4 下一步实验

标注至少 200 个事件的每日阶段，由多名标注者给出潜伏/成长/高潮/消退/再激活；报告 macro-F1、转移违规率、阶段切换延迟和不同领域表现。

## 十三、可疑信息风险评估

### 13.1 当前实现

风险规则包含来源可追溯性、文章级跨平台佐证、官方媒体、传播速度、互动异常、负面情感、煽动性、广告引流、外链、标题正文一致性等连续特征。结果保存 risk score、level、规则版本、特征分、证据和限制。LLM 只在中间灰区结构化复核，不能覆盖显式规则证据。

### 13.2 科学性

把“负面”“传播快”“有外链”直接等同于虚假是不科学的。当前实现已将这些信号降为风险特征，并明确输出“可疑信息风险评估”，这是正确修正。

仍然存在根本限制：没有接入权威事实库、官方辟谣库、知识图谱或证据检索，无法验证命题真假。经验权重可能对自然灾害、政治、金融、医疗和娱乐事件产生不同偏差。LLM 复核也可能受到提示和已有文本偏见影响。

### 13.3 合理性与完整性

作为运营筛查和“需要进一步核验”的排序信号，当前实现合理；作为自动事实核查则不合理。工程上特征、证据、限制和 LLM 灰区比较完整。

### 13.4 下一步实验

建立分领域的文章—主张—证据标注集，至少区分真实、虚假、误导、缺乏证据和无法判断。报告 PR-AUC、固定召回下 precision、各领域假阳性率和校准误差；高风险结果必须链接到可审查证据。

## 十四、传播路径证据图

### 14.1 当前实现

传播图优先使用 parent/root/retweet/quote 等结构化字段和明确转发文本。缺少显式关系时，推断边要求父节点更早、语义达到门槛，并具有来源提及、共享实体、标题覆盖或跨平台跟进等附加证据；综合置信度达标才连边。证据不足时允许有节点无边，并返回 coverage status、limitations 和 evidence。

### 14.2 科学性

传播关系是有方向的因果/引用关系，关键词共现或时间接近不能证明传播。当前“宁可空边、不用主题关系冒充传播”的原则科学且重要。

推断边仍不是真实转发链。即使文章更早且语义相似，也可能来自共同新闻源而非前者影响后者。没有平台原始父子 ID 时，只能称为“可能的跟进/扩散证据”。

### 14.3 完整性

节点角色、传播阶段、证据类型、限制说明和空图状态完整。缺少真实转发链数据、边级评价和共同来源建模。

### 14.4 下一步实验

采集包含明确父子关系的微博/社交转发样本，标注有向文章对；报告显式边覆盖率、推断边 precision/recall、方向错误率，并将“共同来源”作为单独关系类型。

## 十五、AI 元数据、问答与报告

### 15.1 当前实现

事件发布或成员实质变化时生成结构化元数据，保存 status、version、confidence、evidence article IDs 和更新时间。事件详情只读取持久化结果。时间字段优先使用确定性首发时间，LLM 冲突只记录 warning。问答使用事件标题、摘要、生命周期、情感和报道上下文。报告可生成 HTML。

### 15.2 科学性

结构化 JSON、证据文章 ID、确定性字段优先和失败不覆盖旧值，能降低 LLM 随机性造成的破坏。把 LLM 用于摘要、实体补全和解释生成是合理场景。

但“证据文章 ID 存在”不等于生成句子被证据严格蕴含。当前仍可能出现抽取错误、概括过度和问答幻觉。`/ask/stream` 尚不是真正逐 token 流式，PDF 导出也没有充分实证，因此完整性只能评为 3。

### 15.3 下一步实验

为元数据字段建立 evidence precision、字段准确率和遗漏率；问答使用答案可支持性、引用正确率和拒答率；报告进行事实一致性人工评审。引入按句引用或检索片段约束，而不是只提供整体上下文。

## 十六、任务、缓存、持久化与 API 契约

### 16.1 当前实现

后台任务包含 pending/running/success/failed、租约 token、心跳、超时恢复、幂等复用和进度。分析运行、文章特征、事件聚合、情感、热度、生命周期、AI 元数据和 Top10 均持久化。事件详情和查询接口尽量只读，不因浏览触发 LLM 或数据库写入。

API 使用统一响应结构，认证和管理员权限有测试。正式热点与原始 Top10 分离。迁移同时覆盖 MySQL SQL 和 SQLite/Python 路径。

### 16.2 科学性与合理性

这部分属于软件工程而非统计算法。租约、幂等键、缓存和版本化结果是保证算法结果可复现、可追踪的必要条件。测试中发现并修复的一次性验证数据库句柄问题，也说明生命周期管理需要真实环境验证，而不能只依赖内存数据库测试。

### 16.3 完整性

当前工程闭环较强，但仍缺多进程/多实例压力测试、任务队列容量监控、长期数据库增长策略、真实 MySQL 并发验证和完整可观测性。

### 16.4 下一步实验

在 MySQL 环境运行并发任务压测和故障注入：进程中断、租约超时、重复提交、第三方超时、数据库短暂不可用。记录任务完成率、重复执行率、恢复时间和 P95 API 延迟。

## 十七、从用户角度看当前数据质量

### 17.1 用户可以信任的内容

- 页面展示的数据来自真实后端模型和持久化记录，而非主要依赖 mock；
- 今日 Top10 可以解释来源名次和更新时间；
- 正式事件与原始热榜条目有明确区别；
- 情感、生命周期、热度和传播结果包含状态、证据或限制；
- 第三方失败不会静默伪装成成功；
- 事件详情读取不会临时触发新的 LLM 结果。

### 17.2 用户需要看到的警示

- 热度是系统内部可解释排序，不是绝对社会影响力；
- 生命周期是当前阶段估计；
- 风险评估不是事实核查结论；
- 传播推断边不是平台确认的转发关系；
- LLM 生成字段需要结合证据文章阅读；
- 数据覆盖受平台接口、时间窗口和关键词限制。

### 17.3 当前最容易造成误解的地方

如果前端只显示一个分数或阶段名称，而不显示数据量、状态、更新时间、来源和 limitations，用户会把经验算法当成客观事实。成熟展示应优先呈现“结果 + 证据 + 置信度/状态 + 限制”，不能只呈现漂亮图表。

## 十八、优先改进路线

### 18.1 P0：工程与可信展示

1. 为爬虫、LLM 和任务增加成功率、延迟、错误类别监控；
2. 所有前端分析卡展示计算时间、数据量、状态和限制；
3. 验证真实 MySQL、多进程调度和任务恢复；
4. 完成真正 SSE 流式问答与 PDF 导出验证；
5. 为第三方返回结构变化增加契约报警；
6. 建立数据保留、快照清理和结果版本迁移策略。

### 18.2 P1：标注数据与经验参数校准

| 模块 | 标注单位 | 建议规模 | 核心指标 |
|---|---|---:|---|
| 聚合 | 文章对 + 事件簇 | 2,000 对、200 簇 | Pairwise F1、B-cubed、ARI |
| 情感 | 文章标签/分数 | 3,000 篇以上 | Macro-F1、Brier/ECE |
| 生命周期 | 事件—日期—阶段 | 200 个事件 | Macro-F1、转移违规率、切换延迟 |
| 风险 | 文章主张、证据和风险 | 2,000 条以上 | PR-AUC、领域假阳性率 |
| 传播 | 有向文章对 | 1,000 对以上 | Edge precision/recall、方向错误率 |
| Top10 | 每日候选相关性排序 | 30～90 天 | nDCG@10、覆盖率、重复率 |
| 关键词 | 文章关键词/实体 | 1,000 篇 | Precision@5、Recall@10、nDCG |

### 18.3 P2：研究型增强

1. 事件聚合学习阈值或使用有监督 pair scorer；
2. 领域情感模型与概率校准；
3. 多峰生命周期和再激活建模；
4. 热度 learning-to-rank，但保留可解释特征；
5. 使用真实转发链训练时序图模型；
6. 风险模块接入权威辟谣、公告和证据检索；
7. LLM 输出使用按句检索引用和可验证声明分解。

## 十九、不应对外宣称的能力

在完成 E4/E5 评估前，不应使用以下宣传：

- “系统能准确判断新闻真假”；
- “系统能还原真实传播路径”；
- “生命周期预测准确率达到某值”；
- “情感识别适用于所有领域”；
- “热度分数代表真实社会影响力”；
- “事件聚合不会误合并或漏合并”；
- “LLM 报告内容完全可靠”；
- “所有平台爬虫长期稳定可用”。

更合适的表述是：

- 可疑信息风险辅助筛查；
- 基于当前证据的传播关系推断；
- 基于当前时间序列的生命周期阶段估计；
- 多模型、可降级的情感分析；
- 可解释的事件热度排序；
- 具有证据和限制说明的智能分析辅助系统。

## 二十、最终评价

项目当前的主要优势不是某个单一模型特别先进，而是逐步形成了以下成熟特征：

- 原始数据、分析运行和正式事件分层；
- 确定性规则、统计模型、语义模型和 LLM 各自承担合适职责；
- 外部服务失败有降级和诊断；
- 结果有版本、证据、快照和持久化；
- API 与任务状态有回归测试；
- 对无法证明的传播、风险和 LLM 结论开始主动返回限制。

主要短板也很明确：参数仍以经验为主，评估数据集中于合成测试和少量案例，缺少跨领域标注集、长期线上监控和人工一致性评审。

因此，本项目可以评价为：

> **工程完整度已达到较成熟课程项目/可运行原型水平；核心算法选择大体科学且经过明显纠偏，但真实世界有效性仍处于待系统标注验证阶段。**

后续最有价值的投入不是继续堆叠更多模型名称，而是建立数据集、指标、基线、分领域误差分析和持续回归，让经验参数逐步转化为可测量、可复现的证据。

## 二十一、主要参考思想

- Salton/Buckley：TF-IDF 与信息检索权重；
- Blei、Ng、Jordan：Latent Dirichlet Allocation；
- Reimers/Gurevych：Sentence-BERT 语义句向量；
- Cormack、Clarke、Buettcher：Reciprocal Rank Fusion；
- Gomez-Rodriguez、Leskovec、Krause：传播网络推断及其数据要求；
- 信息检索常用 nDCG、Precision/Recall；
- 聚类常用 B-cubed、ARI；
- 概率分类常用 Brier score 和校准误差。

这些理论只能说明方法有依据，不能替代本项目自己的数据评估。
